# Docs Sync

Use this skill after every meaningful code change.

Read:
- AGENTS.md
- docs/PRODUCT.md
- docs/ARCHITECTURE.md
- docs/ITERATION_LOG.md

Rules:
- document reality, not intention
- update ITERATION_LOG every time
- update PRODUCT and ARCHITECTURE when assumptions changed
- keep docs concise and actionable

At minimum include:
- what changed
- affected files
- schema changes
- decisions
- open risks
- next step
